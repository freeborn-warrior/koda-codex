# In-phase consultation

When Producer needs information before a phase artifact can be completed, it
writes a bound consultation request and sends it only to the persistent Reviewer.
The request names the active session, phase, question, evidence inspected, and
suggested authority: Reviewer for evidence or technical judgment, owner for a
product decision.

Reviewer answers from evidence when authorized or discusses the question with the
owner in the Reviewer window. Every actionable response is written as a named
consultation artifact before Producer uses it. Consultation never becomes a
formal review, approval, receipt, product ruling, or phase advancement.

Direction introduced after phase entry is recorded immediately but waits for the
next successful gate. The current phase contract remains frozen. Explicit pushed
halt is the only interrupt; pause-inject-resume is forbidden.
