# Sealed independent review contract — integrated Producer-role preflight

- Written: 2026-07-20
- Review mode: fresh, read-only Codex context
- Intended reviewer: GPT-5.6 Terra, medium effort
- Verdict vocabulary: `APPROVE` or `REVISE`
- This contract must be committed before the review begins.

## Review object

Judge the complete repaired journey, not isolated functions:

1. Quick Start prepares the bundled project and confirmed launch.
2. Before printing `READY`, the installed Codex role sandbox runs the exact first
   Producer `session new` command in a disposable clone.
3. Every live Producer and Reviewer turn automatically carries verified toolkit
   proof plus the narrow Node and Git toolchains.
4. Child Git configuration is isolated without exposing ambient credentials,
   home configuration, sibling projects, network, or write access outside the
   active project.
5. The private XDG directory cannot weaken immutable Guide-return archival.
6. The changed behavior is covered by executable tests and honest failure
   records, not only source assertions.

## Required checks

- Trace the actual call chain from Quick Start and both live role scripts through
  permission arguments and child environment.
- Confirm the exact toolkit evidence is verified before permission is granted.
- Confirm the Quick Start scratch run cannot create an active session in the
  prepared human-demo project.
- Confirm cleanup is bounded to the temporary scratch directory.
- Confirm native Git and XDG changes do not reintroduce ambient secrets, broad
  home access, network, user rules, or a login shell.
- Confirm archival rejects a non-empty or unsafe XDG scratch path.
- Inspect the failed and passing local transcripts and distinguish assertion
  coverage from the installed-client evidence described in the result.
- Identify stale public documentation, test totals, manifest claims, or any
  statement that overclaims a fresh owner-visible pass.

## Verdict rule

`APPROVE` only if the complete journey is internally coherent, fail-closed, and
accurately documented. Use `REVISE` for any concrete correctness, containment,
archive-integrity, regression, or evidence-claim defect. Cosmetic preferences do
not justify `REVISE`.

The review must cite file paths and evidence. A vague blocking verdict is not a
valid finding.
