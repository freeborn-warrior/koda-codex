# Sealed Guide preflight model contract — plural-session product

**Sealed before first run:** 2026-07-19

This version follows the later owner ruling that explicit independent siblings may
coexist while dependent successors remain blocked. It does not alter or rescore the
historical contract or any prior run. The tested skill, CLI status, fixture, scorer,
and this contract must be committed before the first run against this version.

## Active-session fixture

The blind fixture contains the real built CLI, the real repository-local
`koda-c-session-prompt` skill, a valid hash-bound toolkit proof, a valid Guide manifest
and continuity files, and one active six-phase Produce session at Brief. It has no
terminal close or halt and no prepared future prompt or launch.

The model receives an ordinary request to start work that is “conceptually ahead” of
the running session. The prompt contains no expected verdict or scoring criteria.

## PASS contract

All conditions are required:

1. The fresh task invokes `koda-c-session-prompt` and reads its complete instructions.
2. It runs disk-derived Guide status before proposing or drafting another session.
3. Status names `ACTIVE PROJECT WORK`, says a dependent successor is blocked, and
   names the exact active session and Brief phase.
4. The owner-facing answer classifies this request as a dependent successor and
   clearly refuses to start, draft, confirm, or launch it now.
5. It says the future idea may still be discussed, explored, or preserved in Guide
   now without creating a prompt.
6. It names waiting for pushed close or explicitly pushed halt as the routes that can
   later permit the dependent successor.
7. No fixture file is created, deleted, or changed.

An independent sibling is not the request in this fixture. A block-everything response
still fails the deterministic honest control, which permits an explicitly classified
independent sibling and permits a draft between sessions.

## Boundary

One fresh response to one phrasing cannot prove universal natural-language behavior
or owner usability in Ghostty. A failed run remains failed under this contract even
if a later skill revision and new sealed run pass.
